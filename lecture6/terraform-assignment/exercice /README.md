# Terraform Assignment: First Webserver (Local)

> Deploy a simple HTML page to a web server locally using Terraform and Docker.

---

## 🎯 Objectives

1. Install Terraform on your machine
2. Create your first webserver (Docker container running nginx)
3. Deploy a simple HTML page
4. Retrieve the link — output the URL to access your page (`http://localhost:8080`)

---

## 🛠️ Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) — ensure Docker daemon is running
- [Terraform](https://developer.hashicorp.com/terraform/install) — installed on your machine
- No cloud account needed — runs entirely on your machine

## 🚀 Step-by-Step

### Step 1 — Install Terraform

**Linux (WSL/Ubuntu):**
```bash
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

**Verify:**
```bash
terraform version
```

---

### Step 2 — Initialize Terraform

```bash
cd terraform-assignment
terraform init
```

This downloads the required providers:
- `kreuzwerker/docker` — manages Docker containers
- `hashicorp/local` — creates local files

---

### Step 3 — Deploy the Webserver

```bash
terraform apply
```

Type `yes` when prompted. Terraform will:
- ✅ Create `index.html`
- ✅ Pull the `nginx:alpine` Docker image
- ✅ Run a container serving your HTML on port `8080`

---

### Step 4 — Retrieve the URL

After `terraform apply` completes:

```
Outputs:

local_url      = "http://localhost:8080"
container_name = "terraform-assignment-webserver"
```

Open **http://localhost:8080** in your browser 🎉

To view outputs again anytime:
```bash
terraform output
terraform output local_url
```

---

## 🧹 Clean Up

Stop and remove the container:
```bash
terraform destroy
```
Type `yes` when prompted.

## 🔧 Troubleshooting

| Issue | Solution |
|---|---|
| `terraform: command not found` | Add Terraform to your PATH or reinstall |
| Docker provider error | Ensure Docker is running (`docker ps`) |
| Port 8080 in use | Change `external` port in `main.tf` or stop the process using 8080 |
| Cannot connect to URL | Wait a few seconds after apply for the container to start |
| WSL: docker not found | Enable WSL integration in Docker Desktop → Settings → Resources → WSL Integration |

## 💡 How It Relates to AWS

This assignment is a **local simulation** of what Terraform does in the cloud:

| Local (This Assignment) | AWS (Cloud) |
|---|---|
| Docker container | EC2 instance |
| `nginx:alpine` image | AMI (Amazon Machine Image) |
| `localhost:8080` | Public IP address |
| `provider "docker"` | `provider "aws"` |
| Free | Costs money per hour |

The same Terraform concepts apply — you just swap the provider!
